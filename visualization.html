Warum Visualisierung?

Visualisierung transformiert Rohdaten in verständliche Muster und Einblicke. Es hilft dabei:

  • Muster zu erkennen - Trends, Cluster oder Zyklen identifizieren
  • Ausreisser zu entdecken - Ungewöhnliche Datenpunkte visualisieren
  • Zusammenhänge zu verstehen - Beziehungen zwischen Variablen erkennen
  • Kommunikation zu verbessern - Komplexe Ergebnisse einfach darstellen

Ein effektiver Workflow für Datenvisualisierung:

  1. Daten bereinigen und vorbereiten
  2. Forschungsfrage definieren
  3. Passenden Visualisierungstyp auswählen
  4. Grafik erstellen und anpassen
  5. Interpretieren und Insights ableiten

Datenexploration

Bevor wir mit der Visualisierung beginnen, ist es entscheidend, ein grundlegendes Verständnis unserer Daten zu entwickeln. Die Datenexploration hilft uns, Probleme zu identifizieren und die richtigen Visualisierungsmethoden auszuwählen.

Daten können verschiedene Probleme aufweisen, die Ihre Visualisierungen und Analysen beeinträchtigen können:

  • Fehlende Werte - Können zu verzerrten Ergebnissen führen
  • Ausreisser - Extreme Werte, die die Interpretation verfälschen können
  • Datentyp-Probleme - Zahlen, die als Text gespeichert sind
  • Skalenunterschiede - Variablen in unterschiedlichen Massstäben

Grundlegende Datenexploration in R

R Console 
# Datenstruktur untersuchen
str(data)
Erklärung: Die str() Funktion zeigt die Struktur des Datensatzes an - welche Variablen enthalten sind, ihre Datentypen und die ersten paar Werte.
R Console 
# Erste Zeilen anzeigen
head(data)
Erklärung: head() zeigt die ersten 6 Zeilen des Datensatzes an, um einen ersten Eindruck von den Daten zu bekommen.
R Console 
# Zusammenfassende Statistiken
summary(data)
Erklärung: summary() berechnet deskriptive Statistiken für alle Variablen im Datensatz (Mittelwert, Median, Min, Max, Quartile).
R Console 
# Fehlende Werte identifizieren
sum(is.na(data))
Erklärung: Dieser Code zählt die Gesamtzahl der fehlenden Werte (NA) im gesamten Datensatz.
R Console 
# Häufigkeiten einer kategoriale Variable
table(data$gender)
Erklärung: table() erstellt eine Häufigkeitstabelle für kategoriale Variablen. Vor dem "$" gibt man den Datensatz an, nach dem "$" die gewünschte Variable.

Plots: Barplot, Boxplot, Histogram

In diesem Abschnitt lernen Sie drei grundlegende Diagrammtypen kennen, die für verschiedene Arten von Daten geeignet sind. Jeder Plot-Typ hilft Ihnen, andere Aspekte Ihrer Daten zu verstehen und zu kommunizieren.

1. Barplot (Balkendiagramm)

Ein Barplot zeigt die Häufigkeiten von verschiedenen Kategorien. Verwenden Sie ihn, wenn Sie darstellen möchten, wie viele Beobachtungen in jeder Kategorie vorkommen - zum Beispiel wie viele Personen jedem Geschlecht angehören oder welcher Beruf wie häufig vertreten ist.

R Console 
ggplot(data, aes(x = gender))

Erklärung: Mit ggplot(data) beginnen wir eine neue Visualisierung und sagen R, welchen Datensatz wir verwenden möchten. aes(x = gender) definiert, dass die Geschlechtsvariable auf der x-Achse dargestellt werden soll. Die y-Achse wird automatisch als Anzahl der Beobachtungen pro Kategorie berechnet.

R Console 
+ geom_bar(fill = "#b9aadf")

Erklärung: geom_bar() ist die Funktion, die tatsächlich die Balken erstellt. Sie zählt automatisch, wie viele Beobachtungen in jeder Kategorie vorhanden sind. fill = "#b9aadf" färbt alle Balken in einem einheitlichen Lila-Ton. Sie können jeden beliebigen Farbcode verwenden oder Farbnamen wie "blue", "red" etc.

R Console 
+ labs(title = "Anzahl von TeilnehmerInnen nach Geschlecht", 
       x = "Geschlecht", y = "Häufigkeit")

Erklärung: Die labs() Funktion (kurz für "labels") ermöglicht es uns, alle Textbeschriftungen der Grafik anzupassen. Hier setzen wir einen aussagekräftigen Titel, der den Inhalt der Grafik beschreibt, sowie klare Beschriftungen für beide Achsen. Eine gute Beschriftung ist essentiell, damit andere verstehen, was die Grafik darstellt.

So sieht ein bollständiger Code für Barplot aus:

R Console 
# Barplot für Anzahl TeilnehmerInnen nach Geschlecht
ggplot(data, aes(x = gender)) + 
  geom_bar(fill = "#b9aadf") +
  labs(title = "Anzahl von TeilnehmerInnen nach Geschlecht", 
       x = "Geschlecht", y = "Häufigkeit")

Was zeigt uns dieser Barplot?

Dieser Barplot visualisiert die Verteilung der Studienteilnehmer nach Geschlecht. Jeder Balken repräsentiert eine Geschlechtskategorie, und die Höhe des Balkens zeigt an, wie viele Personen dieser Kategorie angehören. In unserem Beispiel sehen wir deutlich, dass mehr Frauen als Männer an der Studie teilgenommen haben.

2. Boxplot

Ein Boxplot ermöglicht den Vergleich von Verteilungen zwischen verschiedenen Gruppen. Er ist ideal, um zu untersuchen, ob sich numerische Werte (wie Zufriedenheitswerte, Alter oder Einkommen) zwischen Kategorien (wie Geschlecht, Berufsgruppen oder Standorten) unterscheiden.

R Console 
# Boxplot für Arbeitszufriedenheit nach Geschlecht
ggplot(data, aes(x = gender, y = job_satisfaction)) + 
  geom_boxplot(fill = "#B6CEB4") +
  labs(title = "Arbeitszufriedenheit nach Geschlecht", 
       x = "Geschlecht", y = "Arbeitszufriedenheit")

Erklärung: Im Gegensatz zum Barplot benötigt der Boxplot zwei Variablen in der aes() Funktion: x = gender für die kategorische Gruppierungsvariable und y = job_satisfaction für die numerische Variable, deren Verteilung wir vergleichen möchten. geom_boxplot() erstellt dann für jede Kategorie auf der x-Achse einen eigenen Boxplot. Die Füllfarbe hilft, die Boxplots visuell voneinander zu unterscheiden.

Wie lese ich einen Boxplot?

Ein Boxplot zeigt fünf wichtige statistische Kennwerte auf einen Blick:

  • Median (die dicke Linie in der Box): Zeigt den mittleren Wert - die Hälfte aller Werte liegt darüber, die andere Hälfte darunter
  • Box (der farbige Bereich): Enthält die mittleren 50% der Daten, vom 25. bis zum 75. Perzentil
  • Whisker (die Linien oberhalb und unterhalb der Box): Zeigen den Bereich der "typischen" Werte, die nicht als Ausreisser gelten
  • Punkte ausserhalb der Whisker: Mögliche Ausreisser - extreme Werte, die deutlich vom Rest der Daten abweichen

In unserem Beispiel sehen wir, dass die Arbeitszufriedenheit bei beiden Geschlechtern ähnlich verteilt ist. Die Mediane liegen nahe beieinander, und die Boxen überlappen sich stark. Dies deutet darauf hin, dass es in dieser Stichprobe keine grossen geschlechtsspezifischen Unterschiede in der Arbeitszufriedenheit gibt.

3. Histogram

Ein Histogramm zeigt die Verteilung einer einzelnen numerischen Variable. Es hilft zu verstehen, wie die Werte einer Messung verteilt sind - ob sie eher niedrig, hoch, gleichmässig oder in Clustern auftreten. Interessanterweise können wir für Histogramme sowohl geom_histogram() als auch geom_bar() verwenden, da beide die Häufigkeit von Werten darstellen.

R Console 
# Histogram für Arbeitszufriedenheit mit geom_bar()
ggplot(data, aes(x = job_satisfaction)) + 
  geom_bar(fill = "#BBDCE5") +
  labs(title = "Verteilung der Arbeitszufriedenheit", 
       x = "Arbeitszufriedenheit", y = "Häufigkeit")

Erklärung: Warum funktioniert geom_bar() auch für numerische Daten? Wenn wir eine numerische Variable wie job_satisfaction auf der x-Achse verwenden, behandelt ggplot2 diese diskreten numerischen Werte (1, 2, 3, 4, 5) ähnlich wie Kategorien. geom_bar() zählt automatisch, wie oft jeder einzelne Wert vorkommt und erstellt dafür Balken. Dies ist besonders praktisch bei Skalen mit wenigen ganzzahligen Werten, wie es bei Zufriedenheitsskalen typisch ist. Für kontinuierliche Daten mit vielen verschiedenen Werten wäre geom_histogram() die bessere Wahl, da es die Werte in Intervalle gruppiert.

Was verrät uns das Histogramm?

Dieses Histogramm zeigt uns die Verteilung der Arbeitszufriedenheitswerte über alle Teilnehmer hinweg:

  • Jeder Balken repräsentiert einen bestimmten Zufriedenheitswert und seine Höhe zeigt, wie viele Personen genau diesen Wert angegeben haben
  • Die Form der Verteilung gibt Aufschluss über die allgemeine Stimmung: Konzentrieren sich die Werte auf der linken Seite (niedrige Zufriedenheit), rechts (hohe Zufriedenheit) oder sind sie gleichmässig verteilt?
  • Besondere Muster wie zwei Gipfel (bimodale Verteilung) können auf unterschiedliche Untergruppen in den Daten hinweisen

In unserer Grafik sehen wir, dass die Werte 3 und 4 am häufigsten vorkommen. Es gibt vergleichsweise wenige Personen mit sehr niedriger Zufriedenheit (Wert 0 oder 1), was auf eine insgesamt positive Grundstimmung hindeutet. Die Verteilung ist rechtsschief, was bedeutet, dass die Tendenz zu höherer Zufriedenheit geht.

Ästhetische Anpassungen

Wir passen Titel, Farben und Positionen an, um Grafiken klarer zu gestalten.

Alles am Plot ist austauschbar!

Lieber eine andere Farbe? Grössere Schrift? Mit ggplot2 kannst du jedes Element eines Plots individuell so anpassen, dass er aufgeräumt und stilvoll wirkt.

Eine effektive Visualisierung ist nicht nur korrekt, sondern auch ästhetisch ansprechend und leicht verständlich.

Schritt 1: Grundgerüst erstellen

Zuerst erstellen wir ein einfaches Balkendiagramm für unsere Daten:

R Console 
# Einfaches Balkendiagramm erstellen
ggplot(data, aes(x = profession)) +
  geom_bar()

Schritt 2: Farben hinzufügen

So kannst du Farben hinzufügen, um die Balken ansprechender zu gestalten:

R Console 
# Balkendiagramm mit einer Farbe
ggplot(data, aes(x = profession)) +
  geom_bar(fill = "#b9aadf")  # Lila Farbe für alle Balken

# Balkendiagramm mit mehreren Farben
ggplot(data, aes(x = profession, fill = profession)) + # Mit "fill = profession" plotest du eine Farbe pro Beruf
    geom_bar() +
    scale_fill_brewer(palette = "Blues")
    
# Balkendiagramm mit Farben nach Geschlecht aufgeteilt
ggplot(data, aes(x = profession, fill = gender)) + 
    geom_bar(position = "dodge") +  # "dodge" = nebeneinander
    scale_fill_brewer(palette = "Blues") + # Das ist optional hier kannst du deine Farbpalette aussuchen
    theme_minimal()

Schritt 3: Titel und Achsen beschriften

Titel und Achsenbeschriftungen machen die Grafik verständlich:

R Console 
# Mit Titel und Achsenbeschriftungen
ggplot(data, aes(x = profession)) +
  geom_bar() +
  labs(
    title = "Anzahl der Beschäftigten nach Beruf",
    x = "Beruf",           # X-Achse beschriften
    y = "Anzahl"           # Y-Achse beschriften
  )

Schritt 4: Textgrössen anpassen

Mit theme() können wir die Textgrössen anpassen:

R Console 
# Textgrössen anpassen
ggplot(data, aes(x = profession)) +
  geom_bar() +
  labs(
    title = "Anzahl der Beschäftigten nach Beruf",
    x = "Beruf",
    y = "Anzahl"
  ) +
  theme(
    plot.title = element_text(size = 16, face = "bold"),  # Titel grösser und fett
    axis.title = element_text(size = 12),                 # Achsentitel Grösse
    axis.text = element_text(size = 10)                   # Achsentext Grösse
  )

Schritt 5: Hintergrund anpassen

Wir können den Hintergrund der Grafik vereinfachen mit theme_minimal():

R Console 
# Komplette Grafik mit allen Anpassungen
ggplot(data, aes(x = profession, fill = profession)) +
  geom_bar() + #Basis des Balkendiagrams
  labs(
    title = "Anzahl der Beschäftigten nach Beruf",
    x = "Beruf", # X-Achsen-Beschriftung
    y = "Häufigkeit", # Y-Achsen-Beschriftung
    fill = "Profession"  # Legendentitel 
  ) +
  theme(
    plot.title = element_text(size = 16, face = "bold"),  # Titel-Stil
    axis.title = element_text(size = 12), # Achsen-Titel-Stil
    axis.text = element_text(size = 10), # Achsen-Titel-Stil
    ) +
    theme_minimal() # Hintergrund vereinfachen
