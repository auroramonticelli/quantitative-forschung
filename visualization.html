Datenexploration & Visualisierungsgrundlagen

Bevor wir mit der Visualisierung beginnen, ist es entscheidend, ein grundlegendes Verständnis unserer Daten zu entwickeln. Die Datenexploration hilft uns, Probleme zu identifizieren und die richtigen Visualisierungsmethoden auszuwählen.

Warum Datenprüfung wichtig ist

Daten können verschiedene Probleme aufweisen, die Ihre Visualisierungen und Analysen beeinträchtigen können:

  • Fehlende Werte - Können zu verzerrten Ergebnissen führen
  • Ausreißer - Extreme Werte, die die Interpretation verfälschen können
  • Datentyp-Probleme - Zahlen, die als Text gespeichert sind
  • Skalenunterschiede - Variablen in unterschiedlichen Maßstäben

Grundlegende Datenexploration in R

Bevor Sie visualisieren, sollten Sie diese grundlegenden Funktionen verwenden:

R Console 
# Datenstruktur untersuchen
str(data)

# Erste Zeilen anzeigen
head(data)

# Zusammenfassende Statistiken
summary(data)

# Fehlende Werte identifizieren
sum(is.na(data))
                  
# Häufigkeiten anschauen
# Vor dem "$" Zeichen, gibt man sein Datenset an
# Nach dem "$" Zeichen, gibt man die gewünschte Variable angeben
# deren Häufigkeit man wissen möchte 
table(data$gender)

Warum Visualisierung?

Visualisierung transformiert Rohdaten in verständliche Muster und Insights:

  • Muster erkennen - Trends, Cluster oder Zyklen identifizieren
  • Ausreißer entdecken - Ungewöhnliche Datenpunkte visualisieren
  • Zusammenhänge verstehen - Beziehungen zwischen Variablen erkennen
  • Kommunikation verbessern - Komplexe Ergebnisse einfach darstellen

Der Visualisierungsprozess

Ein effektiver Workflow für Datenvisualisierung:

  1. Daten bereinigen und vorbereiten
  2. Forschungsfrage definieren
  3. Passenden Visualisierungstyp auswählen
  4. Grafik erstellen und anpassen
  5. Interpretieren und Insights ableiten

Plots: Barplot, Boxplot, Histogram

Wir erstellen verschiedene Plottypen, um Daten zu erkunden.Tipp

Die Auswahl des richtigen Diagrammtyps hängt von der Art Ihrer Daten und der Frage ab, die Sie beantworten möchten:

Barplot (Balkendiagramm)

Für kategorische Daten: Zeigt Häufigkeiten oder Mittelwerte für verschiedene Kategorien.

R Console 
# Barplot für Zylinderanzahlen
ggplot(data, aes(x = factor(gender))) + 
  geom_bar(fill = "#b9aadf") +
  labs(title = "Anzahl von TeilnehmerInnen nach Geschlecht", 
       x = "Geschlecht", y = "Anzahl")

Boxplot (Boxplot)

Für Verteilungen: Zeigt Median, Quartile und Ausreißer einer Verteilung.

R Console 
# Boxplot für MPG nach Zylinderzahl
ggplot(data, aes(x = factor(gender), y = job_satisfaction)) + 
  geom_boxplot(fill = "#b9aadf") +
  labs(title = "Arbeitszufriedenheit Verteilung nach Geschlecht", 
       x = "Geschlecht", y = "Arbeitszufriedenheit")

Histogram (Histogramm)

Für Verteilungen kontinuierlicher Daten: Zeigt die Häufigkeitsverteilung einer Variable.

R Console 
# Histogram für MPG
ggplot(data, aes(x = job_satisfaction)) + 
  geom_histogram(fill = "#b9aadf", bins = 10, color = "white") +
  labs(title = "Verteilung der Arbeitszufriedenheit", 
       x = "Arbeitszufriedenheit", y = "Häufigkeit")
R Console 
# Barplot
ggplot(data, aes(x = factor(cyl))) + geom_bar(fill = "#b9aadf")

# Boxplot
ggplot(data, aes(x = factor(cyl), y = mpg)) + geom_boxplot(fill = "#b9aadf")

# Histogram
ggplot(data, aes(x = job_satisfaction)) + geom_histogram(fill = "#b9aadf", bins = 10)

Ästhetische Anpassungen

Wir passen Titel, Farben und Positionen an, um Grafiken klarer zu gestalten.Tipp

Eine effektive Visualisierung ist nicht nur korrekt, sondern auch ästhetisch ansprechend und leicht verständlich. ggplot2 bietet umfangreiche Anpassungsmöglichkeiten:

Gute Praxis für Visualisierungen

  • Klare Titel und Beschriftungen - Jede Grafik sollte selbsterklärend sein
  • Angemessene Skalierung - Vermeiden Sie irreführende Darstellungen
  • Farben mit Bedeutung - Verwenden Sie Farben konsistent und zugänglich
  • Vermeiden von Chartjunk - Entfernen Sie unnötige decorative Elemente
  • Hervorhebung wichtiger Punkte - Lenken Sie die Aufmerksamkeit auf Schlüsselelemente

Die Anpassung des Erscheinungsbilds erfolgt in ggplot2 hauptsächlich durch das theme()-Argument und verschiedene Skalenfunktionen.

R Console 
ggplot(mtcars, aes(x = hp, y = mpg)) +
  geom_point(color = "#b9aadf") +
  geom_smooth(method = "lm", se = FALSE, color = "#9b7bb5") +
  labs(
    title = "Horsepower vs. MPG",
    subtitle = "Beispiel mit angepasster Schriftgröße",
    x = "Horsepower (HP)",
    y = "Miles per Gallon (MPG)"
  ) +
  theme(
    plot.title = element_text(size = 20, face = "bold", color = "#9b7bb5"),
    plot.subtitle = element_text(size = 16, face = "italic", color = "#b9aadf")
  )
