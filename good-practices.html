Gute Gewohnheiten beim Programmieren

Eine der wichtigsten Gewohnheiten beim Programmieren in R ist das Schreiben von gut dokumentiertem und nachvollziehbarem Code. Dies erleichtert nicht nur die Zusammenarbeit mit anderen, sondern hilft auch Ihnen selbst, wenn Sie nach einiger Zeit zu Ihrem Code zurückkehren.

Warum gute Gewohnheiten wichtig sind:

  • Code wird leichter verständlich und nachvollziehbar
  • Erleichtert die Fehlersuche (Debugging)
  • Ermöglicht die Wiederverwendung von Code
  • Fördert die Reproduzierbarkeit von Analysen
  • Spart langfristig Zeit und Mühe

R Code: Kommentare und schrittweises Vorgehen

R Terminal 
#Das ist ein Kommentar
              
#So kannst du erklären was du machst
Beispiel für gut kommentierten und strukturierten R-Code

Tipps für gutes Kommentieren:

  • Verwenden Sie Kommentare, um Abschnitte zu kennzeichnen (# 1. Abschnitt ----)
  • Erklären Sie den Zweck komplexerer Code-Blöcke
  • Halten Sie Kommentare kurz und prägnant
  • Aktualisieren Sie Kommentare, wenn Sie den Code ändern
  • Verwenden Sie aussagekräftige Variablennamen, die Kommentare teilweise überflüssig machen

R-Skripte verwenden

Ein R-Skript ist eine Textdatei mit der Endung .R, die R-Code enthält. Anstatt Code direkt in der Konsole einzugeben, sollten Sie ihn in Skripten speichern. Dies hat mehrere Vorteile:

Vorteile von R-Skripten:

  • Code ist dauerhaft gespeichert und kann später wieder verwendet werden
  • Einfaches Teilen von Code mit anderen
  • Nachvollziehbare und reproduzierbare Analysen
  • Einfacheres Debugging und Testen von Code
  • Versionierung mit Tools wie Git möglich

R Code: Struktur eines typischen R-Skripts

R Terminal 
# Titel: Datenanalyse der Umfrage zur Arbeitszufridenheit
# Autor: Aurora Monticelli
# Datum: 2025-09-24
# Beschreibung: Dieses Skript analysiert die Umfragedaten zur Arbeitszufriedenheit von
# Gesundheitsfachpersonen in Basel-Stadt und Basel-Land 

# 1. Pakete laden ---------------------------------------
library(tidyverse)  # Sammlung von Paketen für Datenwissenschaft

# 2. Daten importieren ---------------------------------
# Daten aus CSV-Datei laden
data <- read.csv("Desktop/healthcare_job_satisfaction.csv")

So arbeiten Sie mit R-Skripten:

  • Öffnen Sie RStudio und erstellen Sie ein neues Skript (File > New File > R Script)
  • Schreiben Sie Ihren Code im Skript-Editor
  • Führen Sie Codezeilen mit Strg+Enter (Windows) oder Cmd+Enter (Mac) aus
  • Speichern Sie Ihr Skript regelmäßig (mit der Endung .R)
  • Kommentieren Sie Code aus, den Sie vorübergehend deaktivieren möchten

Daten importieren

Der Import von Daten ist oft der erste Schritt in jeder Datenanalyse. R bietet verschiedene Möglichkeiten, Daten aus unterschiedlichen Quellen und Formaten zu importieren.

Häufig verwendete Datenformate:

  • CSV (Comma-Separated Values) - weit verbreitetes Textformat
  • Excel (.xlsx, .xls) - Tabellenkalkulationsdateien
  • SPSS (.sav), SAS (.sas7bdat), Stata (.dta) - Formate statistischer Software
  • JSON - Format für strukturierte Daten
  • Datenbanken - SQLite, MySQL, PostgreSQL etc.

R Code: Daten aus verschiedenen Quellen importieren

R Terminal 
# 1. Aus CSV-Datei importieren
data <- read.csv("Desktop/healthcare_job_satisfaction.csv")

Tipps zum Datenimport:

  • Überprüfen Sie immer die Struktur der importierten Daten mit str()
  • Stellen Sie sicher, dass Dezimaltrennzeichen und Spaltentrennzeichen korrekt erkannt werden
  • Beachten Sie die Encoding-Einstellungen bei Umlauten (UTF-8 wird empfohlen)
  • Definieren Sie relative Pfade statt absoluter Pfade für bessere Portabilität
  • Speichern Sie importierte Daten als RData-Format für schnellere spätere Verwendung

Pakete laden und installieren

R-Pakete sind Sammlungen von Funktionen, Daten und Dokumentation, die die Funktionalität von R erweitern. Die Fähigkeit, Pakete zu installieren und zu laden, ist entscheidend für effektives Arbeiten mit R.

Wichtige R-Pakete für Einsteiger:

  • tidyverse - Sammlung von Paketen für Datenwissenschaft (dplyr, ggplot2, etc.)

R Code: Pakete installieren und laden

R Terminal 
# 1. Pakete installieren
install.packages("tidyverse")   # Einzelnes Paket

# 2. Pakete laden
library(tidyverse)     # Paket laden

Bewährte Praktiken für Pakete:

  • Laden Sie nur die Pakete, die Sie tatsächlich benötigen
  • Aktualisieren Sie Pakete regelmäßig mit update.packages()
  • Verwenden Sie library() am Anfang Ihres Skripts
  • Notieren Sie die verwendeten Pakete in der Dokumentation
  • Beheben Sie Konflikte zwischen Paketen mit package::function()
