R Studio installieren

Bevor du mit R programmieren kannst, musst du R und R Studio installieren. R ist die Programmiersprache selbst, während R Studio eine open source software ist, die das Arbeiten mit R erleichtert.

Besuche posit.co und lade die kostenlose R Studio Desktop Version herunter. Führe die heruntergeladenen Installationsdateien aus und folge den Anweisungen.

Die verschiedenen Fenster in R Studio

Nachdem du R Studio geöffnet hast, siehst du vier Hauptbereiche:

R Studio Desktop

1. Quelltext-Editor (Source Editor)

Dies ist der Bereich, in dem du deine R-Skripte schreibst und bearbeitest. Hier kannst du Code schreiben, speichern und organisieren. Der Vorteil gegenüber der direkten Eingabe in der Konsole ist, dass du deine Arbeit speichern und später wieder verwenden kannst.

2. Konsole (Console)

In der Konsole wird R-Code ausgeführt und die Ergebnisse werden angezeigt. Du kannst Code direkt in der Konsole eingeben, aber es ist besser, Code im Quelltext-Editor zu schreiben und dann zur Konsole zu senden, um ihn auszuführen.

3. Umgebung/Verlauf (Environment/History)

Dieser Bereich zeigt alle aktuell definierten Variablen, Daten und Funktionen. Im Verlaufsteil siehst du alle zuvor ausgeführten Befehle. Dies ist besonders nützlich, um den Überblick über deine Arbeit zu behalten.

4. Dateien/Plots/Pakete/Hilfe (Files/Plots/Packages/Help)

Dieser Bereich hat mehrere Tabs:

  • Files: Zeigt die Dateien in deinem Arbeitsverzeichnis
  • Plots: Zeigt erstellte Grafiken
  • Packages: Listet installierte Pakete auf und ermöglicht deren Verwaltung
  • Help: Bietet Zugang zur R-Dokumentation und Hilfeseiten

Was ist ein R-Code?

In R ist Code eine Reihe von Anweisungen, die du dem Computer gibst, um bestimmte Aufgaben auszuführen. Diese Anweisungen werden in der R-Sprache geschrieben und folgen einer bestimmten Syntax (Grammatik der Programmiersprache).

Code in R kann verschiedene Zwecke haben:

  • Daten manipulieren: Daten sortieren, filtern, transformieren
  • Berechnungen durchführen: Statistische Analysen, mathematische Operationen
  • Visualisierungen erstellen: Grafiken, Diagramme, Plots
  • Funktionen definieren: Eigene wiederverwendbare Code-Blöcke erstellen
R Terminal 
# Einfache Berechnung - das ist Code!
5 + 3

# Eine Variable erstellen - das ist Code!
meine_zahl <- 10

# Eine Funktion aufrufen - das ist Code!
mean(c(1, 2, 3, 4, 5))

Gute Gewohnheiten beim Programmieren

Eine der wichtigsten Gewohnheiten beim Programmieren in R ist das Schreiben von gut dokumentiertem und nachvollziehbarem Code. Dies erleichtert nicht nur die Zusammenarbeit mit anderen, sondern auch das spätere Wiederauffinden und Verstehen des eigenen Codes.

Warum gute Gewohnheiten wichtig sind:

  • Code wird leichter verständlich und nachvollziehbar
  • Erleichtert die Fehlersuche (Debugging)
  • Ermöglicht die Wiederverwendung von Code
  • Fördert die Reproduzierbarkeit von Analysen

R Code: Kommentare und schrittweises Vorgehen

R Terminal 
#Das ist ein Kommentar
              
#So kannst du erklären was du machst
Beispiel für gut kommentierten und strukturierten R-Code

Tipps für gutes Kommentieren:

  • Verwende Kommentare, um Abschnitte zu kennzeichnen (# 1. Abschnitt ----)
  • Erkläre den Zweck komplexerer Code-Blöcke
  • Halten Kommentare kurz und prägnant
  • Aktualisiere Kommentare, wenn der Code geändert wird

R-Skripte verwenden

Ein R-Skript ist eine Textdatei mit der Endung .R, die R-Code enthält. Anstatt Code direkt in der Konsole einzugeben, ist es besser, ihn in Skripten zu speichern. Dies hat mehrere Vorteile:

  • Code ist dauerhaft gespeichert und kann später wieder verwendet werden
  • Einfaches Teilen von Code mit anderen
  • Nachvollziehbare und reproduzierbare Analysen
  • Einfacheres Debugging und Testen von Code

R Code: Struktur eines typischen R-Skripts

R Terminal 
# Titel: Datenanalyse der Umfrage zur Arbeitszufridenheit
# Autor: Aurora Monticelli
# Datum: 2025-09-24
# Beschreibung: Dieses Skript analysiert die Umfragedaten zur Arbeitszufriedenheit von
# Gesundheitsfachpersonen in Basel-Stadt und Basel-Land 

# 1. Pakete laden ---------------------------------------
library(tidyverse)  # Sammlung von Paketen für Datenwissenschaft

# 2. Daten importieren ---------------------------------
# Daten aus CSV-Datei laden
data <- read.csv("Desktop/healthcare_job_satisfaction.csv")

Deine ersten Schritte mit einem R-Skript:

  • Öffne RStudio – deine neue Programmier-Heimat.
  • Klicke oben auf File > New File > R Script. !
  • Vor dir erscheint ein leeres Fenster. Das ist deine Schaltzentrale, in die du deinen Code schreibst.
  • Speichere dein Skript (z.B. als mein_erstes_script.R). So geht keine Arbeit verloren!

Daten importieren

Der Import von Daten ist oft der erste Schritt in jeder Datenanalyse. R bietet verschiedene Möglichkeiten, Daten aus unterschiedlichen Quellen und Formaten zu importieren.

R Code: Daten aus verschiedenen Quellen importieren

R Terminal 
# 1. Aus CSV-Datei importieren
data <- read.csv("Desktop/healthcare_job_satisfaction.csv")

Pakete laden und installieren

R-Pakete sind Sammlungen von Funktionen, Daten und Dokumentation, die die Funktionalität von R erweitern. Die Fähigkeit, Pakete zu installieren und zu laden, ist entscheidend für effektives Arbeiten mit R.

Das tidyverse ist eine Sammlung von R-Paketen, die speziell für die Datenwissenschaft entwickelt wurden. Für Beginner ist es der ideale Einstieg, weil:

  • Alle Pakete folgen den gleichen Prinzipien und Syntax-Regeln
  • Die Funktionen haben konsistente und leicht verständliche Namen
  • Es deckt den gesamten Datenanalyse-Workflow ab: Import, Aufbereitung, Visualisierung und Modellierung
  • Die Dokumentation und Community-Unterstützung ist ausgezeichnet

Zu den wichtigsten Paketen im tidyverse gehören dplyr für Datenmanipulation und ggplot2 für Grafiken.

R Code: Pakete installieren und laden

R Terminal 
# Paket installieren (nur einmal notwendig)
install.packages("tidyverse")

# Paket laden (in jeder R-Session notwendig)
library(tidyverse)

Wichtige Hinweise zur Paketverwaltung:

  • Pakete müssen nur einmal installiert (install.packages()) werden, aber in jeder neuen R-Session geladen werden (library())
  • Das Laden aller Pakete am Anfang eines Skripts ist empfehlenswert
  • Eine Dokumentation (library()) der verwendeten Pakete erleichtert die Reproduzierbarkeit
