Gute Gewohnheiten beim Codieren

Beim Einstieg in R wirkt vieles komplex. Eine einfache, aber sehr hilfreiche Gewohnheit ist das Verwenden von Kommentaren. Kommentare beginnen mit einem # und werden von R ignoriert. Sie dienen ausschließlich zur Erklärung des Codes. So wird deutlich, welcher Schritt gemacht wird und warum er wichtig ist. Dadurch bleibt der Code auch nach längerer Zeit verständlich.

R Console 
# Schritt 1: Daten laden
# Ein Datensatz wird von der Festplatte eingelesen
daten <- read.csv("meine_daten.csv")

R-Skripte

In R wird Code meist in Skripten geschrieben. Ein Skript ist eine einfache Textdatei mit der Endung .R. Der Vorteil: Code kann gespeichert, jederzeit wieder ausgeführt und schrittweise verbessert werden. So entsteht eine Art „Rezeptbuch“ für Analysen, das immer wieder nutzbar bleibt.

R Console 
library(ggplot2) 
daten <- read.csv("meine_daten.csv")
ggplot(daten, aes(x = Variable1, y = Variable2)) +
  geom_point()

Daten & Pakete laden

Vor einer Analyse müssen zunächst die benötigten Pakete installiert und geladen werden. Pakete sind Zusatzsammlungen von Funktionen – vergleichbar mit Apps auf einem Smartphone. Danach werden die Daten importiert, meist aus einer CSV-Datei. Erst dann kann mit den eigentlichen Analyseschritten begonnen werden.

R Console 
install.packages("tidyverse")
library(tidyverse)
daten <- read.csv("meine_daten.csv")
