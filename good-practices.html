Gute Gewohnheiten beim Codieren

Wenn Sie mit R anfangen, wirkt vieles vielleicht kompliziert. Eine der wichtigsten Gewohnheiten ist: **Schreiben Sie Kommentare**. Ein Kommentar ist eine Zeile, die mit # beginnt. Er wird von R nicht ausgeführt, sondern dient nur als Erklärung für Sie oder andere. So behalten Sie leichter den Überblick, wenn Sie Ihren Code später noch einmal lesen.

R Console 
# 1. Daten laden
# Hier importieren wir unseren Datensatz
daten <- read.csv("meine_daten.csv")

R-Skripte

In R schreiben Sie Ihren Code meistens in **Skripten**. Das sind einfache Textdateien mit der Endung .R. Der Vorteil: Sie können Ihren Code speichern, später erneut ausführen und Schritt für Schritt verbessern. Statt alles immer neu einzutippen, bauen Sie sich so ein wiederverwendbares „Rezept“ für Ihre Analyse.

R Console 
library(ggplot2) 
daten <- read.csv("meine_daten.csv")
ggplot(daten, aes(x = Variable1, y = Variable2)) +
  geom_point()

Daten & Pakete laden

Bevor Sie Daten analysieren können, müssen Sie zwei Dinge tun: 1. **Pakete installieren und laden** – das sind Zusatzfunktionen für R, ähnlich wie Apps auf Ihrem Handy. 2. **Ihre Daten importieren** – z. B. eine CSV-Datei, in der Ihre Tabellen gespeichert sind. Danach können Sie direkt loslegen und die Daten untersuchen.

R Console 
install.packages("tidyverse")
library(tidyverse)
daten <- read.csv("meine_daten.csv")
